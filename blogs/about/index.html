    关于我

    GumplinGo 7/28/2021

    我是郭江林，Gumplin Go， 前端工程师，现于深圳“体验”生活。

    大家好，我是郭江林，英文名是 Gumplin Go , 采用音译，从潮汕话翻译：‘江林’ 是 Gumplin, ‘郭’ 取 Go，我喜欢 Go， 有启程的感觉，所以不是 Kwok，所以就是 Gumplin Go 啦！

    本人前端攻城狮一枚，19届毕业的非科班，靠搜刮网上学习资源成长的野路子程序猿 😅 。

    # 关于本站

    我一直想自己做一个博客，从 0 到 1 那种，但这是一个巨大的目标，这不仅涉及到前端开发, 还涉及到服务端渲染、数据库设计、接口开发等等，我一想到要做评论功能等等这些，就一直拖延逃避。最近看到一篇博文，该博主非科班自学一年就去了大厂。虽然单看学习经历，是和我很相似的，但是结果却大不同，人家去了大厂，且仅用了一年，而我还在温水煮青蛙。我决定奋发图强，先迈出那一步再说。博客先搭起来，文章先写起来，先积累着，尽管是用静态网站生成器搭建起来。

    # 为什么学编程 (why)

    TLTR：当然是编程很酷还高薪啦。

    我一直处于一个被动的成长状态：因为会得到家长老师的认可夸赞，所以就尽力学习；因为分数线尴尬，但同意被调剂可以去一个知名度高一点的学校，就同意调剂了；因为觉得 PS 能做炫酷的东西，就去学习怎么用，尽管不太需要...一直被动，到了大三，面临就业压力，开始迷茫自己要干什么，无所适从。这时，听说编程高薪，决定去学。

    However! 编程是我于被动状态中，作的带着那么一点点主动的、反抗的选择。小时候玩游戏，幻想过自己什么时候能开发一个外挂，为所欲为；高考填志愿，填的全是和计算机、软件相关的专业，尽管我是二指禅，除了开关机，其他几乎都不懂；大学对编程的印象就是:高大上，难，和神秘的 hacker，但我依旧做出这样的选择，想要向那神秘的领域探索。

    # 怎么学编程 (how)

    TLTR：入门靠视频，提升靠阅读思考。动手实践验证，纸上得来终觉浅。

    学编程的渠道很多，大多是网上的。我从计算机小白开始，其实没那么顺利。因为 Python 的名头比较大，我一开始是找了本 python 书看，但是，尽管用了高中做语文阅读题的专注力，还是没看懂，门儿都没进。我总结原因是自己菜，其次就是看书而已，缺乏实践。

    这时候了解到，网上挺多视频课程，实战课程之类的，各种夺人眼球的标题，让人热血沸腾啊。比如，某马前端开发毕业班全套，某课网xx实战等等。那时的我，还是学生，经济拮据，就到处寻求免费课程资源。找到的资源也比较多，跟着视频从安装开发环境开始，到跟着一步一步做出一些小 demo, 也算是入门了一点点前端。入门技术不难，难在枯燥，难在坚持。

    我经过“调研”，了解到前端入门容易一些，就学的是前端。至于具体学了什么，我简单罗列一下：HTMl、CSS、Javascript、NodeJs、Git、Vue, 多但一知半解。

    学习方法固然重要，但我觉得不是重点，重点是学习的驱动力有多大，你想要做点什么，那么就会去研究怎么实现。我自己的驱动力其实不算大，有点佛，不像那些“卷王”博主，个个输出爆表，拿到大厂 offer。

    最近看到阮一峰阮老师的周刊一个视频vs读书的学习话题 (opens new window)：

    人类学习效率最高的方式，不是"读书 + 思考"，而是"观察 + 模仿"。 大部分领域的入门方式，都是"观察和模仿"，所以视频学习更有效。

    我比较认同：我一开始就是啃书，吃力不讨好；但是后面想要提升了，得看书。视频相对来说就比较浪费时间了，一个知识点看书很快理解，但是视频就得耐着性子慢慢听。

    # 哪里学编程 (where)

    • BiliBili (opens new window) 上面搜索关键字，会有你需要的
    • 各项技术都有自己文档，都有自己的 “Getting Started” 章节, 印记中文 (opens new window) 收录翻译了很多前端技术文档，既能查看，又可以作为学习路线图，说到学习路线，Github (opens new window) 上面也有很多的 roadmap
    • 慕课网、极客时间、开课吧等等，虽说这些不少是付费课程，倡导支持正版，但你有心，还是可以找到流传的资源啦
    • 博客网站：CSDN、掘金、知乎等等, 国外的有 DEV community (opens new window)
    • 国外的在线学习网站：Udemy (opens new window) 也是不错的
    • Google (opens new window), 善用搜索引擎 ...

    # 其他

    关于英语：大学考了好几次终于压线过的英语六级，让我对英语有本能的排斥。然而，我第一份工作的同事，是个英语很棒的人，把生活布满英文环境。我秉着见贤思齐的念头，也克服了对英语的排斥。这对编程其实有好处: 技术知识文档，还是英文的比较新鲜，代码出问题也是英文报错。

    关于打字：说到这点，你可能会笑，打字有啥好提的。其实我大学专业论文都是二指禅敲出来的，我也烦恼怎么才能打字快。问了朋友，都是说打多了就快，更有甚者，说你打游戏，被气的想骂人就会快了。好像有道理，但没用啊。我决定必须克服这个问题，不然妨碍我写 bug 的速度啊！于是找到金山打字，严格按照里面设置的规则和关卡，每天练习半小时，直到能盲打。后面还了解到双拼这输入法，也学习了。说这点，其实是想说明，你想做什么，你就得针对着去做，直到完成。编程亦是如此，你要完成什么效果，需要什么技能，就去学。绝知此事要躬行！

    感谢光临本站，有什么疑问，欢迎留言交流啊 ❤️ ❤️~
